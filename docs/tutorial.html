§1这是什么

装上这个插件后，你可以在 ClassIsland 的自动化里加两类新 行动。它们做的事情非常直白：

弹一个窗口，让用户拍板，再决定后面跑什么。

两个 Action 长这样：

「询问窗」 —— 弹窗含固定「执行 / 取消」两按钮，「执行」打开预配置的文件/程序/文件夹。
「多按钮询问」 —— 弹窗可配 N 个按钮，按哪个就走哪个按钮自己的 Action 链。

关键特性：弹窗是 置顶 的，完全禁止通过 X 关闭（最小化会被自动还原），必须通过按钮才能走。所以你写好的自动化不会因为学生随手关窗而漏跑。

§2我能用它做什么

下面是三个最常见的场景。具体字段含义见 §4。

询问窗

下课自动打开作业系统

"下课事件触发 → 弹窗『是否打开今天的作业系统？』→ 点执行 → 自动打开 作业系统 URL。"

多按钮询问

课前选今天的学习模式

"上课事件触发 → 弹窗『听写 / 默写 / 自习』三选一 → 按哪个就走哪个 Action 链。"

多按钮 + 自动执行

晚自习结束做点什么

"21:30 弹窗『整理书包 / 写日记 / 跳过』→ 30 秒不点就 自动按「整理书包」。"

§3怎么装

  1. 打开 GitHub Releases，下载最新的 InquiryWindow.cipx 文件。
  2. 打开 ClassIsland 主界面 → 设置 → 插件 → 安装本地插件。
  3. 选择刚才下载的 .cipx 文件，重启 ClassIsland（完全退出再打开）。
  4. 打开任一自动化的 行动集 编辑器，右键 → 添加行动，搜"询问"应能看到两个。
注意 · platform

当前版本只支持 Windows。自动提取 .exe 图标依赖 System.Drawing.Common，Linux/macOS 上行为未测试。

§4两个 Action + 自动执行怎么选

两种 Action 共享"自动执行"功能。先看总表：

name
trigger
fit for
A
询问窗Inquiry Window
弹窗含固定「执行 / 取消」。按"执行"则打开预配置的 .exe / 文件 / 文件夹 / URL。
"要打开某个东西吗？点执行就开。"
B
多按钮询问Multi-button Prompt
弹窗含 1~N 个按钮（每按钮自带 Action 链）。按哪个就走哪个链，期间不再询问。
"今天要干啥？三选一。"
C
自动执行Auto Execute
弹窗显示 N 秒倒计时 + 进度条。归零自动按指定目标（按钮 / 无事发生）。
"等一会没人点就按默认走。"

窗口标题StringOS 标题栏文字

在弹窗中显示目标路径bool在正文下方用等宽字体预览

窗口标题StringOS 标题栏
主提示Markdown主标题，Markdown 渲染

模式 C · 自动执行 —— 详细字段（两个 Action 共享）

勾上设置页下方的 「在弹窗显示后自动触发」 复选框后展开。默认关闭。

● auto-execute2~3 properties

下拉框最后一项 「无事发生」 是合法占位：到时间后不执行任何 Action，仅自动关闭弹窗。适合"等 N 秒用户不点就跳过"的场景。

§5举个完整的栗子

假设你想做一个"下课自动打开作业系统，30 秒不点就自动执行"的自动化：

目标 · goal

下课事件触发 → 弹窗 『是否打开今天的作业系统？』 → 用户点"执行"立即打开；30 秒不点就自动执行。

步骤

  1. 新建自动化，触发器选 下课事件。
  2. 添加行动 「询问窗」，填：标题=下课啦、正文=是否打开今天的作业系统？、目标路径=https://your-homework-system。
  3. 勾上 「在弹窗显示后自动执行」，秒数设 30。
  4. 保存，启用自动化。

效果

✅ 下课时弹窗，含"将0分30秒后执行......"倒计时和进度条。
✅ 用户点"执行"立即打开目标，倒计时被取消。
✅ 用户点"取消"或最小化（被自动还原），30 秒后窗口自动关闭并执行。

注意 · caveat

① 自动执行的目标按下拉索引记录。如果你重排了按钮顺序，目标会指向错误位置 —— 改完顺序请重新选一次。

② 「目标路径」为空时倒计时不会启动（没东西可执行）。

③ 询问窗是 Windows-only，Action 抛异常会弹错误对话框，但不影响后续 Action 链。

§6常见问题

装上后怎么没看到行动？›

先确认 ClassIsland 已完全重启（不是最小化、是真的退出再打开）。然后去任一自动化的行动编辑器里 右键 → 添加行动，搜"询问"应能看到两个。

支持哪些变量？›

询问窗的标题/正文里可写：

{time} · 当前时间（HH:mm）
{date} · 当前日期
{subject} · 当前课程
{nextSubject} · 下一节课程

变量找不到值时显示为空，不会报错。

倒计时归零了但 Action 还没跑完窗口就关了？›

正常。多按钮询问：倒计时归零 → 触发目标按钮 → 窗口立即关闭，Action 链在后台跑。Action 链执行过程不再弹任何询问窗。如果链里某步失败，错误会写日志但不影响后续步骤。

怎么改按钮图标？›

设置页按钮 Expander 里有 「图标字符」 输入框，填 Fluent 图标库的字符码（十六进制）。比如 \uE73E 是 √，\uE711 是 ✕。

不确定码值时点旁边的 「…」 按钮可以从图标库挑。

「无事发生」什么时候用？›

想"等 N 秒用户不点就 跳过整个弹窗"的场景。比如：上课前弹窗问"是否进入听写模式"，30 秒不点就按"无事发生"——直接关窗、不执行任何 Action、流程继续往下走。

按钮被删了自动执行会乱吗？›

不会。删除目标按钮时下拉框会自动回退到第一个真实按钮；如果所有按钮都被删完且自动执行已启用，弹窗仍会出现，倒计时结束后按"无事发生"自动关闭（这一步会写日志告警）。

倒计时文本不显示？›

先检查两件事：

① 设置页 「在弹窗显示后自动触发」 复选框是否勾上；

② 询问窗需要 目标路径非空 才会启动倒计时（多按钮询问则无此限制）。

怎么反馈问题？›

去 GitHub 仓库的 Issues 页面提，记得附上 ClassIsland 版本、插件版本和日志（看 %AppData%\ClassIsland\Logs\ 下的最新日志）。
